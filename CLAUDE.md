## Repository Structure

Monorepo — two services, one repo:
- `api/` — Rails 8.1 API-only (Ruby 3.3.6, PostgreSQL)
- `web/` — React 19 + TypeScript (Node 23.6, Vite)

Local Postgres runs via Docker: `docker compose up -d`

## API Commands

Run from `api/`:

```bash
docker compose up -d                         # Start Postgres + MailCatcher (run once, from repo root)

bin/rails server                             # Start dev server (port 3000)
bin/rails console
bin/rails db:migrate
bin/rails db:seed                            # Creates admin@example.com
bin/rails test                               # Run all tests
bin/rails test test/models/foo_test.rb       # Run a single test file
bin/ci                                       # Full local CI — rubocop, brakeman, bundler-audit, tests
bin/rubocop                                  # Lint only (rubocop-rails-omakase)
bin/brakeman                                 # Security scan only
bin/bundler-audit check --update             # Dependency vulnerability scan only
```

## Web Commands

Run from `web/`:

```bash
npm run dev      # Start Vite dev server (port 5173)
npm run build    # TypeScript check + Vite build
npm run lint     # ESLint
```

## Auth Architecture

**Do not change this without understanding the full pattern.**

- Login/signup → API returns `{ token, user }` in JSON body + sets `session_token` httpOnly cookie
- Token stored in `AuthContext` (React memory only — never localStorage)
- `tokenStore` singleton bridges AuthContext to the api client module
- Page refresh → `AuthContext` calls `GET /api/v1/session` on mount → Rails reads cookie → returns token
- Any 401 → `api/client.ts` silently retries refresh → retries original request → calls `onUnauthenticated` if refresh fails
- `ProtectedRoute` component handles redirect to `/login` for unauthenticated users
- `require_admin` before_action available in any controller for admin-only endpoints

Key files: `web/src/context/AuthContext.tsx`, `web/src/api/client.ts`, `api/app/controllers/concerns/authentication.rb`

## API Endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/api/v1/session` | No | Login |
| GET | `/api/v1/session` | Cookie | Silent refresh |
| DELETE | `/api/v1/session` | Yes | Logout |
| POST | `/api/v1/users` | No | Register |
| GET | `/api/v1/me` | Yes | Current user |
| POST | `/api/v1/passwords` | No | Request password reset |
| PATCH | `/api/v1/passwords/:token` | No | Reset password |

## Email in Development

All outgoing email is caught by MailCatcher — nothing is actually sent. Start it with `docker compose up -d` alongside the database, then open `http://localhost:1080` to see all emails (password reset links, etc.) in a web UI.

If MailCatcher is not running and an email is triggered, Rails will raise an error (intentional — fail loudly in dev rather than silently dropping mail).

## Rails Conventions

- Controllers live in `app/controllers/api/v1/`
- Serializers use Blueprinter — `UserSerializer.render_as_hash(record)` returns a Hash, `UserSerializer.render(record)` returns a JSON string. Use `render_as_hash` inside `render json:`
- All errors return `{ error: "..." }` (single) or `{ errors: [...] }` (validation)
- `Current.user` is available in any authenticated controller action
- rubocop-rails-omakase style: spaces inside array brackets `[ :a, :b ]`

## Frontend Conventions

- Tailwind CSS v4 — no config file, configured via `@tailwindcss/vite` plugin
- shadcn/ui components in `src/components/ui/` — add with `npx shadcn add <component>`, style: `base-nova`, base color: `neutral`
- Path alias `@/` maps to `src/`
- Icons via `lucide-react`
- `routeTree.gen.ts` is auto-generated by TanStack Router Vite plugin — do not edit manually
- Add new routes as files in `src/routes/` — the plugin picks them up automatically
